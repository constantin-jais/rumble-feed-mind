# Preuve locale Radar — synchronisation publique bornée — 2026-07-13

## Portée de la preuve

Cette traversée locale importe deux abonnements OPML publics, fetch les flux sous allowlist HTTPS exacte, inspecte 20 items, produit un `CuratedItemExport`, l’embarque dans le bundle Dioxus puis rejoue la revue sur Chromium, Firefox et WebKit à 390 × 844.

Elle prouve une exécution locale réelle du parcours `source → signal → qualification → décision → export`. Elle ne vaut ni disponibilité publique, ni release, ni approbation humaine, ni déploiement. Le manifeste généré porte explicitement `publication_authorized: false`.

Les hashes de contenu changeront légitimement lorsque les flux publics évoluent. Le script recrée `target/live-radar-proof/` depuis zéro et ne publie aucun artefact.

## Limites restantes

- import et synchronisation déclenchés par CLI avant build, pas depuis l’interface ;
- pas de scheduler, compte, API de curated review ou persistance durable multi-appareil ;
- résolution DNS prévalidée mais non pinnée atomiquement pendant la connexion ;
- preuve locale non revue humainement ; Radar reste `discovery` et sa gate publique demeure fermée.
